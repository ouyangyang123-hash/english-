# 英语口语自学系统 · Vlog 字幕 + Notion

## 1. 系统概述

从真实 Vlog 口语语料中学习地道英语，而非课本英语。

```
YouTube/Vlog → yt-dlp → SRT字幕 → Claude Code分析
                                            ↓
    ┌───────────────────────────────────────┘
    ↓
Notion 词汇银行 ←── SRS 间隔重复 ──→ Claude Code 每日测试
    ↓
每日签到页 → 周/月回顾 → 持续进步
```

| 工具 | 用途 |
|------|------|
| **yt-dlp** | 下载 YouTube 字幕 |
| **Claude Code** | 字幕分析、词汇提取、SRS调度、练习互动 |
| **Notion** | 词汇数据库、SRS参数存储、进度追踪仪表盘 |
| **Python** (可选) | 批量处理多个视频字幕 |

## 2. 字幕提取流水线

### 2.1 工具安装

```bash
# 安装 yt-dlp
winget install yt-dlp
# 或
pip install yt-dlp

# 安装 ffmpeg（如需提取音频）
winget install ffmpeg

# 验证安装
yt-dlp --version
```

### 2.2 YouTube 字幕提取

```bash
# 方式1: 自动生成字幕（大多数视频可用）
yt-dlp --write-auto-subs --sub-lang en --skip-download \
  --output "./subtitles/%(title)s" \
  "https://youtube.com/watch?v=VIDEO_ID"

# 方式2: 手动上传字幕（质量更高）
yt-dlp --write-subs --sub-lang en --skip-download \
  --output "./subtitles/%(title)s" \
  "https://youtube.com/watch?v=VIDEO_ID"

# 方式3: 先查看有哪些字幕语言
yt-dlp --list-subs "https://youtube.com/watch?v=VIDEO_ID"

# 批量下载整个播放列表
yt-dlp --write-auto-subs --sub-lang en --skip-download \
  --output "./subtitles/%(playlist_index)s-%(title)s" \
  "PLAYLIST_URL"
```

### 2.3 本地 SRT 文件清洗

将 `.srt` 文件拖入 Claude Code，执行清洗：

```
请帮我处理这个 SRT 字幕文件：./subtitles/my-vlog.en.srt

执行以下步骤：
1. 移除所有时间戳行和序号行
2. 合并相邻的连续台词
3. 移除 [Music]、[Applause]、[inaudible] 等非对话标记
4. 去除重复的无意义填充词（如连续出现3次以上的 "um"/"uh"）
5. 将清理后的纯文本保存到 ./subtitles/my-vlog-clean.txt
6. 统计词频（去除 stop words），列出前50个高频词
```

### 2.4 一键深度分析（核心工作流）

```
请对这段 YouTube Vlog 字幕进行完整的英语学习分析。

YouTube URL: [填入链接]

请自动完成：
1. 使用 yt-dlp 下载英文字幕（--write-auto-subs --sub-lang en --skip-download）
2. 清洗提取纯文本
3. 按话题自动分类（Travel / Food / Tech / Daily Life / Emotions / Work / Social / Health）
4. 提取所有 B1/B2/C1 级别的词汇（排除 A1-A2 基础词），格式如下：
   | 词汇 | 中文释义 | 原文例句 | 话题分类 | 难度 |
5. 提取所有短语动词 (phrasal verbs) 和地道搭配 (collocations)
6. 提取所有习语和俚语表达 (idioms & slang)
7. 标注出值得背诵的"黄金句型"（地道、高频、实用的句子）
8. 生成一份 Notion 可导入的 CSV 格式词汇表
```

### 2.5 批量处理脚本（Python 骨架）

```python
# batch_extract.py — 批量下载字幕
# 用法: python batch_extract.py urls.txt ./subtitles/

import subprocess
import sys
import os

def extract_subs(url, output_dir):
    """下载单个视频的英文字幕"""
    cmd = [
        "yt-dlp",
        "--write-auto-subs", "--sub-lang", "en",
        "--skip-download",
        "--output", f"{output_dir}/%(title)s.%(ext)s",
        "--no-playlist",
        url.strip()
    ]
    result = subprocess.run(cmd, capture_output=True, text=True)
    if result.returncode != 0:
        print(f"  FAIL: {url.strip()}")
        return False
    return True

def main():
    output_dir = sys.argv[2] if len(sys.argv) > 2 else "./subtitles"
    os.makedirs(output_dir, exist_ok=True)

    with open(sys.argv[1], "r", encoding="utf-8") as f:
        urls = [line.strip() for line in f if line.strip()]

    print(f"Processing {len(urls)} videos...\n")
    success, fail = 0, 0
    for i, url in enumerate(urls, 1):
        print(f"[{i}/{len(urls)}] {url}")
        if extract_subs(url, output_dir):
            success += 1
        else:
            fail += 1

    print(f"\nDone. Success: {success}, Failed: {fail}")

if __name__ == "__main__":
    main()
```

## 4. 间隔重复系统 (SRS)

### 4.1 SM-2 算法

```
初始状态:
  EF (简易度系数) = 2.5
  Interval (间隔天数) = 1
  n (复习次数) = 0

每次复习后，用户评分 q (0-5):
  5 = 完美回忆，毫不费力
  4 = 犹豫后正确
  3 = 费力但正确
  2 = 错误，但看答案后想起
  1 = 错误，看答案后感觉熟悉
  0 = 完全遗忘

── 评分 ≥ 3（回忆成功）──
  若 n = 0: Interval = 1 天
  若 n = 1: Interval = 6 天
  若 n ≥ 2: Interval = round(Interval × EF)
  n += 1

── 评分 < 3（回忆失败）──
  n = 0
  Interval = 1 天
  （重置该词汇的复习进度）

── 更新 EF ──
  EF' = EF + (0.1 - (5-q) × (0.08 + (5-q) × 0.02))
  若 EF' < 1.3，则 EF' = 1.3

下次复习日 = 今天 + Interval 天
```

### 4.2 SM-2 速查表

| 当前n | q=5 | q=4 | q=3 | q=2/1/0 |
|-------|-----|-----|-----|---------|
| n=0 | I=1, n=1 | I=1, n=1 | I=1, n=1 | I=1, n=0 |
| n=1 | I=6, n=2 | I=6, n=2 | I=6, n=2 | I=1, n=0 |
| n=2, EF=2.5 | I=15, n=3 | I=15, n=3 | I=15, n=3 | I=1, n=0 |
| n=3, EF=2.5 | I=38, n=4 | I=38, n=4 | I=38, n=4 | I=1, n=0 |
| n=4, EF=2.5 | I=95, n=5 | I=95, n=5 | I=95, n=5 | I=1, n=0 |

> 注：实际间隔 = round(I × EF)，取决于当前的 EF 值。上表假设 EF 保持 2.5。

### 4.3 Claude Code SRS 每日复习

```
请帮我完成今日的 SRS 词汇复习。

从 Notion 词汇银行中筛选「下次复习日 ≤ 今天」的所有词汇。
按简易度系数(EF)从低到高排序，优先复习最难记的词汇。

测试流程（对每个词汇）：
1. 显示「中文释义」+「词性」，让我说出/拼写英文
2. 根据我的回答，严格按照 SM-2 标准评分 (0-5)
3. 显示正确答案和原文例句
4. 根据评分计算：新 EF、新间隔天数、新复习次数、下次复习日
5. 输出更新参数，格式如下：
   [词汇] q=X / EF: 2.5→2.6 / I: 1→6天 / n: 0→1 / 下次: 2026-06-04

全部复习结束后输出统计：
📊 今日复习总结
- 复习总数：__个
- 正确率：__% (q≥3)
- 平均评分：__分
- 🔴 需重点关注的词汇（q≤2 的词汇列表）
```

### 4.4 每日复习工作流

```
步骤1  打开 Notion「📋 今日待复习」视图，确认待复习数量
步骤2  在 Claude Code 中运行 SRS 每日复习提示词
步骤3  Claude 逐词测试你，每题后给出 SRS 更新参数
步骤4  将更新参数回填到 Notion（EF、间隔、下次复习日等）
步骤5  在「每日签到页」记录今日完成情况
```

## 5. 练习模式

### 5.1 闪卡测试 (Flashcard)

```
闪卡模式 · 从 Notion 词汇银行随机抽取20个词。

规则：
1. 显示中文释义，等我说出英文
2. 正确 → ✅，进入下一词
3. 错误 → ❌，进入「纠错循环」
   纠错循环：该词进入队尾，重新测试直到连续2次正确
4. 全部结束后输出：
   📊 闪卡成绩单
   - 总词数：20
   - 一轮正确：__词 (__%)
   - 最终正确率：__%
   - 🔴 困难词清单（纠错≥3次的词，建议加入今日重点复习）
   - 正确率数据可直接填入 Notion 签到页
```

### 5.2 填空练习 (Cloze Deletion)

```
填空练习模式 · 10题。

请从 Notion 词汇银行中，选取最近7天学过的词汇。
为每个词汇生成原文例句的填空题版本（目标词替换为 _____）。

规则：
1. 一次展示一道题
2. 如需要，输入「提示」获得首字母提示
3. 输入「跳过」放弃本题，直接看答案
4. 填完后判断正误并展示完整句子
5. 结束输出：
   📊 填空成绩单
   - 正确：__/10
   - 跳过的词汇（建议复习）：[列表]
```

### 5.3 中译英翻译 (Translation)

```
中译英翻译模式 · 10题。

从词汇银行中抽取10个 🟢已掌握 或 🟡学习中 的词汇。
为每个词汇生成一个中文句子，翻译时必须用到指定词汇。

规则：
1. 出示：中文句子 + 必须使用的英文词汇
2. 我提交英文翻译
3. 检查三个维度：
   ✓ 是否使用了指定词汇
   ✓ 整体语法是否正确
   ✓ 表达是否地道自然
4. 给出评分（1-5）+ 改进建议 + 参考译文

全部结束后输出成绩单。
```

### 5.4 听写练习 (Dictation)

```
听写练习模式。

请从视频素材库中选取一段字幕（约30秒长度的文本）。

流程：
1. 🎧 第一遍：正常语速朗读整段（整体理解，不动笔）
2. 🎧 第二遍：逐句慢速朗读，每句重复2遍，等我写完再下一句
3. 🎧 第三遍：正常语速再读一遍（检查确认）
4. 📝 展示原文，和我写的逐句对比
5. 🔍 分析：
   - 漏听/误听的词汇
   - 不熟悉的连读/弱读现象
   - 建议加入词汇银行的生词
```

### 5.5 综合测验（30分钟）

```
综合测验模式 · 30分钟完整练习。

🕐 时间安排：
Round 1 · 闪卡热身 (5分钟)
  随机10词，快速过一遍

Round 2 · 填空练习 (10分钟)
  10题 Cloze，基于最近视频字幕

Round 3 · 中译英 (10分钟)
  5句翻译，指定使用本周重点词汇

Round 4 · 总结 (5分钟)
  错误分析 + 学习建议 + Notion 更新清单
```

### 7.4 常见问题

**Q: yt-dlp 下载字幕失败（报错 "no subtitles"）？**
A: 该视频可能禁用了字幕。尝试 `--write-auto-subs`（自动生成字幕），大多数英语视频都有。如果仍然失败，换一个视频或手动下载 SRT 文件。

**Q: SRT 文件打开后乱码？**
A: 确保文件编码为 UTF-8。在 VS Code 中打开后，点击右下角编码 → "Reopen with Encoding" → UTF-8，然后保存。

**Q: Notion 数据库关系怎么建立？**
A: 在词汇银行中添加 Relation 属性 → 选择 Videos 数据库。然后在 Videos 中添加 Rollup 属性，选择关联的词汇银行的词汇条目 → Count。

**Q: SM-2 参数更新太麻烦，能自动化吗？**
A: 目前需要手动回填到 Notion。但 Claude Code 每次测试后都会直接告诉你更新后的参数，复制粘贴即可。如果需要完全自动化，可以开发 Notion API 集成脚本。

**Q: 每天应该复习多少个词？**
A: 取决于 SRS 待复习队列。初期每天10-20个，随着词汇量增长，每天可能需要复习30-50个。如果队列过长，优先复习 EF 最低（最难记）的词汇。

**Q: 自动生成字幕的准确度够吗？**
A: 对于词汇挖掘完全够用。即使个别词识别错误，Claude 在分析时会基于上下文修正。如果有条件，优先选择带手动字幕的视频（CC标志）。
